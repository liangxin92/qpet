# Reviewer Notes

## Plugin shape

- Skills-only plugin; no MCP server, OAuth, API key, hooks, telemetry, or developer backend.
- Uses host-provided image generation and the currently installed Codex pet workflow.
- Writes creative outputs to the chosen project and, only after validation/confirmation, a local Codex pet package.

## Independent value beyond the native pet creator

The plugin does not claim that 3D-toy styling alone is unique. Its reviewable value is the pre-production and control layer:

1. deterministic generation of three concepts that differ on at least four design axes
2. host-controlled selfie/reference transformation with a minimal likeness lock
3. reusable `pet-spec.json` identity constraints
4. targeted repair of failed coherent animation states
5. strict package validation, conflict detection, and safe-install rules

The package contains a future-design note for an account-usage HP companion, but the market skill never queries rate limits, starts a monitor, injects a HUD, or advertises that unsupported capability.

## Camera boundary

No script calls a camera, browser, AppleScript, media API, or photo library. A selfie enters only through a visible host camera/photo attachment action initiated by the user. The source image is never added to the final pet package.

## Product boundary

“3D chibi” describes the rendered visual style. Outputs are transparent 2D sprite atlases, not GLB models or real-time 3D. The plugin does not change native pointer events or app states and does not claim cloud sync or hosted one-click installation.

## Review fixtures

`test-cases.json` contains exactly five positive and three negative tests. Tests use original generic concepts and no third-party logos or copyrighted characters.
