# Public Submission Checklist

## Completed in this package

- valid `.codex-plugin/plugin.json`
- one self-contained skill with scripts and references
- market name, short description, long description, category, capabilities, and starter prompts
- release notes and reviewer notes
- privacy policy, terms, and support-page copy
- ready-to-host landing-page copy
- exactly five positive and three negative review tests
- original logo, dark logo, composer icon, and representative concept screenshot
- no API keys, external server, hooks, silent camera access, telemetry, or bundled third-party character assets

## Publisher actions required before submission

These values cannot be invented by the code package:

1. Replace `Petai Studio` with the verified individual/developer display name if different.
2. Complete OpenAI individual or business identity verification.
3. Confirm Apps Management Write permission for the submitting organization.
4. Host a public website, support page, privacy policy, and terms page over HTTPS.
5. Add those real HTTPS URLs to `plugin.json` and the portal fields.
6. Review the included original brand assets and replace them only if desired.
7. Choose supported countries/regions and confirm policy declarations.
8. Run all eight review tests on the required surfaces and save the actual results.
9. Zip the skill/plugin payload exactly as required by the current submission portal.
10. Submit for review; after approval, use the portal's separate publish action.
