# Privacy Policy

Effective date: July 15, 2026

## Scope

This policy covers the Qpet skills-only plugin. The plugin does not operate a developer-owned server, user account system, analytics service, advertising system, or external image database.

## Data used for a request

Depending on what the user chooses to provide, a request may use:

- keywords, style choices, pet names, and creative instructions
- a selfie captured through a visible host camera/photo action
- an image the user deliberately attaches and is authorized to transform
- generated concept images, sprite sheets, previews, and validation reports
- local file paths and pet ids needed to save or install the requested pet

The plugin does not need passwords, API keys, contacts, precise location, microphone access, or the rest of a user's photo library.

## Camera and portrait handling

The plugin cannot and does not silently activate a camera. Camera capture or photo selection is initiated by the user through controls supplied by ChatGPT/Codex or the operating system. Only the selected image is used for the requested transformation.

The workflow uses visible appearance cues needed for the chibi design, such as hair shape, glasses, facial hair, face silhouette, expression, and chosen clothing colors. It is not intended to identify a person, perform face recognition, calculate biometric templates, or infer sensitive traits.

## Processing and sharing

Inputs and generated images are processed by the OpenAI host capabilities the user invokes. This plugin does not send them to an additional developer-operated backend or sell, rent, or share them for advertising. OpenAI's own processing is governed by the terms and privacy controls of the user's OpenAI service.

## Storage and deletion

Working files and finished pets may be saved locally in the user's selected project and Codex pet directories. The source selfie is not included in the finished pet package. The workflow discloses any retained working copy and offers to remove that copy after packaging; the user's original attachment or photo-library item is never deleted by the plugin.

Users can remove generated run folders and locally installed pets from their own device at any time. Because this release has no developer backend, the publisher does not retain a separate server copy to delete.

## Security and support

The plugin validates package paths to reduce path traversal and requires explicit confirmation before replacing an existing pet. For privacy or support questions, contact the publisher through the public support page linked in the plugin directory listing.
